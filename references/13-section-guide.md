# 13섹션 상세페이지 가이드

## 개요
고전환 상세페이지의 13개 섹션 구조와 각 섹션의 역할을 설명합니다.

## 전체 흐름: 감정 여정

```
[주목] → [공감] → [이해] → [희망] → [신뢰] → [확신] → [행동]

01.Hero    : 주목 + 관심
02.Pain    : 공감 + 동질감
03.Problem : 원인 이해
04.Story   : 희망 + 가능성
05.Solution: 해결책 인지
06.How     : 방법 이해
07.Proof   : 신뢰 형성
08.Authority: 전문성 확인
09.Benefits: 가치 인식
10.Risk    : 불안 제거
11.Compare : 선택 압박
12.Filter  : 자격 확인
13.CTA     : 행동 유도
```

## 섹션별 상세

### 01. Hero (긴급성 헤더)
**목적**: 3초 안에 관심 캡처

**필수 요소**:
- 헤드라인: 핵심 혜택 + 구체적 결과
- 서브헤드: 타겟 명시 + 방법 힌트
- CTA 버튼: 명확한 행동 유도
- 긴급성 배지: 한정/희소성 요소

**높이**: 800px

**체크리스트**:
- [ ] 타겟이 "이건 나를 위한 거다" 느끼나?
- [ ] 핵심 혜택이 한눈에 보이나?
- [ ] 지금 행동해야 할 이유가 있나?

---

### 02. Pain (공감 섹션)
**목적**: "이거 내 얘기다" 공감 유발

**필수 요소**:
- 공감 질문: "혹시 이런 고민 있으신가요?"
- 페인포인트 3-4개: 구체적 상황 묘사
- 감정 훅: 마무리 공감 문구

**높이**: 600px

**작성 팁**:
- 추상적 표현 대신 구체적 상황
- "많은" → "매일 3시간씩"
- "힘들다" → "새벽 2시까지 컴퓨터 앞에서"

---

### 03. Problem (문제 정의)
**목적**: 실패 원인 이해 → "내 탓 아님" 안도

**필수 요소**:
- 반전 훅: "~이 안 되는 건 당신 탓이 아닙니다"
- 진짜 원인 3개: 구조적/시스템적 문제
- 관점 전환: 해결책 기대감 유발

**높이**: 500px

**핵심 메시지**:
"당신이 실패한 게 아니라, 기존 방법이 틀렸던 것"

---

### 04. Story (Before→After)
**목적**: 변화 가능성 확신

**필수 요소**:
- Before: 과거의 고통 상태
- 전환점: 무엇이 바뀌었는지
- After: 현재의 성공 상태
- 증거: 구체적 숫자/결과

**높이**: 700px

**스토리 구조**:
```
"저도 6개월 전까지 [Before 상태]였습니다.
하지만 [전환점]을 만난 후,
지금은 [After 상태]입니다."
```

---

### 05. Solution Intro (솔루션 소개)
**목적**: 제품 정체성 명확화

**필수 요소**:
- 제품명: 기억하기 쉬운 이름
- 한 줄 정의: 핵심 가치 요약
- 타겟 적합성: "~를 위한"

**높이**: 400px

---

### 06. How It Works (작동 방식)
**목적**: 쉬워 보이게 + 신뢰 형성

**필수 요소**:
- 3-4단계 프로세스
- 각 단계별: 번호 + 제목 + 설명 + 결과
- 최종 결과 강조

**높이**: 600px

**원칙**:
- 최대 4단계 (더 많으면 복잡해 보임)
- 각 단계 10단어 이내
- 시각적 프로세스 표현

---

### 07. Social Proof (사회적 증거)
**목적**: 신뢰도 최대화

**필수 요소**:
- 숫자 증거: 고객 수, 만족도, 성과
- 실제 후기 3-5개: 이름 + 결과 + 감상
- 미디어/인증: 있다면 추가

**높이**: 800px

**후기 선택 기준**:
1. 구체적 결과가 있는 후기
2. Before→After가 명확한 후기
3. 타겟과 비슷한 사람의 후기

---

### 08. Authority (권위/전문성)
**목적**: 제작자 신뢰 구축

**필수 요소**:
- 소개: 이름 + 직함
- 이력/실적: 관련 경험과 성과
- 진정성 메시지: 왜 이걸 만들었는지

**높이**: 500px

---

### 09. Benefits + Bonus (혜택)
**목적**: 가치 극대화

**필수 요소**:
- 핵심 혜택 리스트 (5-7개)
- 보너스 구성 (가치 명시)
- 총 가치 계산 (정가 대비 할인)

**높이**: 700px

**가치 표현**:
```
✅ 핵심 커리큘럼 (가치 300만원)
🎁 보너스 1: 템플릿 패키지 (가치 50만원)
🎁 보너스 2: 1:1 컨설팅 (가치 100만원)

총 450만원 상당 → 오늘만 49만원
```

---

### 10. Risk Removal (리스크 제거)
**목적**: 구매 장벽 제거

**필수 요소**:
- 환불/보장 정책
- FAQ 3-5개
- 지원 내용

**높이**: 500px

**핵심 FAQ**:
1. "저도 할 수 있을까요?"
2. "시간이 많이 필요한가요?"
3. "효과가 없으면요?"
4. "언제부터 결과가 나오나요?"

---

### 11. Before/After Final (최종 대비)
**목적**: 선택 압박

**필수 요소**:
- Without: 구매 안 하면 (부정적)
- With: 구매하면 (긍정적)
- 선택 질문

**높이**: 400px

**구조**:
```
[이 제품 없이]          [이 제품과 함께]
❌ 계속 헤매며           ✅ 검증된 방법으로
❌ 시간 낭비             ✅ 최단 시간 성과
❌ 불확실한 미래         ✅ 확실한 결과

"어떤 미래를 선택하시겠습니까?"
```

---

### 12. Target Filter (타겟 필터)
**목적**: 적합한 고객만 남기기

**필수 요소**:
- 추천 대상 (이런 분께 추천)
- 비추천 대상 (이런 분은 비추)

**높이**: 400px

**효과**:
- 부적합 고객 필터링 → 환불률 감소
- 적합 고객 확신 → 전환율 증가

---

### 13. Final CTA (최종 CTA)
**목적**: 즉각적 행동 유도

**필수 요소**:
- 마지막 헤드라인
- 가격 (정가 취소선 + 할인가)
- 긴급성 재강조
- CTA 버튼 (크고 눈에 띄게)
- 마무리 문구

**높이**: 600px

**CTA 버튼 문구 예시**:
- "지금 시작하기"
- "50% 할인 받기"
- "무료 체험 신청"
- "바로 구매하기"
